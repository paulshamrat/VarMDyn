# CDKL5 Structural Validation & Annotation Pipeline

This directory (`manuscript/assets/cdkl5_structure_annotation`) serves as the unified computational pipeline used to extract, validate, and graphically annotate the canonical structural elements of the CDKL5 kinase domain.

The outputs here provide the direct graphical and tabular evidence for the structural-annotation panel in **Figure 1B** and for **Supplementary Figure S1/Table S4** in the manuscript.

## 2026-05-18 P-loop Annotation Correction

An annotation audit found that the earlier Figure 1B label treated the CDKL5
glycine-rich/P-loop as `14-21`, while kinase literature supports two related but
non-identical concepts:

- the broader kinase P-loop/G-loop lies between beta1 and beta2;
- the strict CDKL5 glycine-rich `GxGxxG`-type motif is `20-25` (`GEGAYG`).

The active manuscript figure now colors the strict motif and uses the shortened
display label `Gly-rich/P-loop (20-25)` for space. The broader beta1-beta2
region is retained in the structural table/prose as context, with Patel 2010
(`10.1021/pr100662s`), Grant 1997 (`10.1074/jbc.272.27.16946`), and Dar 2011
(`10.1146/annurev-biochem-052410-090317`) as the support trail.

Important lesson: changing SVG text alone is not sufficient for structural
corrections. The green cartoon coloring is baked into the PyMOL-rendered base
image, so any residue-range correction must regenerate the PyMOL base render
first, then regenerate the SVG/PNG and manuscript composite.

## 1. Sequence Alignment & Conservation (MSA)

The scripts in this section computationally prove the evolutionary conservation of CDKL5's structurally defined regions against canonical kinases.

### Scripts:
1. `fetch_sequences.py`: Pulls the 8 required kinase sequences directly from UniProt (CDKL1-5, CDK2, ERK2, PKA-Cα).
2. `make_cdkl_family_msa.py`: Automates the MUSCLE v5 alignment of the fetched sequences.
3. `parse_helix_ranges.py`: Connects sequence boundaries to true physical structure by parsing `HELIX` and `SHEET` records directly from Canning et al. 2018 PDB depositions (e.g. 4BGQ).
4. `build_helix_table.py`: Formats the parsed mapping into the markdown table used for the Supplementary Data.
5. `generate_msa_figure.py`: Uses `matplotlib` to render the final ESPript-style sequence alignment figure showing the physical overlay of $\beta$-sheets and $\alpha$-helices.

## 2. 3D Structure Rendering

These files control the application of the validated coordinates to the 3D CDKL5 structural model.

### Files:
- `cdl.com.wat.leap.pdb`: The ATP--Mg-bound homology model scaffold. (Origin: This file is the minimized starting coordinate frame generated by Amber LEaP from the main MD setup pipeline, incorporating the 4BGQ homology model and the Hu2024 ATP--Mg parameters used for the actual simulations).
- `viz.pml`: The PyMOL script that individually colors every cross-validated secondary structure component (Gly-rich/P-loop motif, $\alpha$C, Catalytic Loop, Activation Loop, CDKL-specific $\alpha$J, etc.) according to the MSA/literature boundaries. It exports the base unannotated image: `cdkl5_wt_atp_mg_final.png`.

## 3. SVG Vector Annotation

To preserve high-quality typography, a Python script applies standard SVG overlays to the rendered PyMOL background.

### Script:
- `make_svg_annotation.py`: Uses `svgwrite` to explicitly place matching colored labels (and a separation line between the N-Lobe and C-Lobe) exactly over the corresponding protein structures. 
- Output: `cdkl5_annotated.svg` and `cdkl5_annotated_preview.png`.

## Final Manuscript Figure Disclaimer

While this pipeline is fully reproducible to the baseline `cdkl5_annotated_preview.png` output, **the figures actively used in the manuscript are manual, user-applied aesthetic modifications** applied on top of the generated vector file.

Specifically, the user took the generated `cdkl5_annotated.svg`, manually refined the layout/aesthetics, and saved their own modified graphics:
- `cdkl5_annotated_mod.svg` (The physically modified vector)
- `cdkl5_annotated_mod.jpg` (An exported JPG variant)
- `cdkl5_annotated_mod.png` (The current active figure embedded directly into `results.tex`)

Executing this pipeline from scratch using the methods below will strictly regenerate the *baseline* SVG and PNG files without overwriting the user's manual modifications.

## Reproducing the Baseline Outputs

```bash
# 1. Fetch, align, and generate MSA evidence
python fetch_sequences.py
python make_cdkl_family_msa.py
python parse_helix_ranges.py
python build_helix_table.py
python generate_msa_figure.py

# 2. Regenerate 3D base structure in PyMOL (headless)
export PYMOL_BIN=/path/to/pymol
"$PYMOL_BIN" -cq viz.pml

# 3. Overlay SVG vector annotations
python make_svg_annotation.py

# 4. Export active manuscript panel from manually refined SVG
/snap/bin/inkscape cdkl5_annotated_mod.svg -o cdkl5_annotated_mod.png -b white -y 1.0
```

After changing the PyMOL base render or active panel PNG, rebuild the manuscript
Figure 1 composite and PDF from the repository root:

```bash
python manuscript/assets/main_candidates/mechanism_rmsf_combined_review/build_split_mechanism_rmsf_review.py
make -C manuscript -B main.pdf
```

## Historical Implementation Plan & Results

To preserve the record of how this pipeline was initially constructed and validated, the original implementation plan is summarized below:

### The Original Plan
**Goal:** Validate all CDKL5 helix/loop/motif residue ranges via literature + MSA, then propagate confirmed ranges into PyMOL, SVG, and the manuscript.
- **Stage 1 (Literature):** Find deposited PDB IDs for CDKL structures (4BGQ) and extract canonical CMGC motifs from literature (Reinhardt2023, Dar2011, Haldane2016).
- **Stage 2 (MSA):** Build an 8-sequence FASTA covering CDKL1-5, CDK2, ERK2, and PKA-Cα. Align with MUSCLE.
- **Stage 3 (Parsing):** Parse the `HELIX` and `SHEET` records directly from the 4BGQ crystal structure and map them to the MUSCLE alignment using Python.
- **Stage 4 (MSA Figure):** Generate an ESPript-style colored alignment figure showing conservation at all labeled motif positions (especially the CDKL-unique $\alpha$J helix).
- **Stage 5 (PyMOL+SVG):** Update PyMOL rendering to explicitly color confirmed ranges. Generate an SVG overlay with color-matched typography.
- **Stage 6 (Manuscript):** Inject the generated MSA figure, structural table, and annotated 3D render into the manuscript supplementary and main results sections.

### Final Results & Manuscript Integration
The pipeline was fully successful and automated. The generated data was integrated into the manuscript as follows:
- **Main Figure (Annotated Structure):** The baseline PyMOL+SVG output was manually refined by the user into `cdkl5_annotated_mod.png`, which is actively compiled into the Figure 1 structural-context composite as **Figure 1B**.
- **Supplementary Table (Structural Boundaries):** The script `build_helix_table.py` was refactored to directly output a raw LaTeX tabular block (`helix_table.tex`), which is dynamically injected via `\input{...}` into `supplementary.tex` as **Table S4**.
- **Supplementary Figure (MSA Conservation):** The generated ESPript-style alignment (`cdkl_family_msa_all.png`) is directly compiled into `supplementary.tex` as **Supplementary Figure S1**.
